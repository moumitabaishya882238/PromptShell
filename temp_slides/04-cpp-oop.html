OOP in the backend

C++ and OOP concepts used here

PromptShell is not a single-file script. It is split into classes so each part has one clear job, which makes the system easier to maintain and extend.
Encapsulation • Abstraction • Polymorphism

Main OOP ideas

Encapsulation

Each class keeps its data and helper methods together, so the rest of the code only uses the public API.

Abstraction

The terminal user sees simple commands, while the internal model and compiler steps stay hidden.

Inheritance

AssistantMode is a base class and the different modes extend it with their own behavior.

Polymorphism

Debug, concise, and code-only modes can be switched without changing the main flow.

Core classes

PromptShellCoordinates the whole prompt, answer, and output workflow.
LLMEngineLaunches llama-cli and reads the model response.
CodeRunnerCompiles and runs C++ code, then captures errors.
AssistantModeChanges behavior for default, concise, code-only, and debug modes.

Why this is good design

Single ResponsibilityEach class does one thing only, so the code is easier to test and fix.
ReusabilityAssistant modes and runner logic can be reused without rewriting the whole app.
MaintainabilityWhen the LLM or compiler flow changes, only the related class needs updates.
ScalabilityFuture features like a Node.js bridge or VS Code extension can plug into this structure.
