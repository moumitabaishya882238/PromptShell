End-to-end working flow

How everything connects

This slide shows the pipeline from writing C++ code to building PromptShell, running the backend, loading TinyLlama through llama.cpp, and returning a clean response.
C++ + Build + Model + Output

1. C++ Source Code

The user writes a file like abc.cpp and passes it to PromptShell for run or debug.

Input
→

2. PromptShell Backend

C++ CLI reads the file, runs compile/run logic, and prepares the prompt for the assistant.

Controller
→

3. CMake + Ninja

CMake generates build files, Ninja compiles fast, and the backend binary is produced.

Build

4. CodeRunner

Compiles the code with g++ and captures compile/runtime output for debug mode.

→

5. llama.cpp

The C++ inference engine calls the local model and generates the explanation.

→

6. TinyLlama GGUF

The actual model weights are loaded from the GGUF file stored in the models folder.

→

7. Clean Response

The answer is post-processed and shown in the terminal for screenshot-friendly output.

Short version

  • C++ handles the app logic.
  • CMake + Ninja handle building.
  • llama.cpp runs the local model.
  • TinyLlama gives the AI response.
  • PromptShell formats the result.
